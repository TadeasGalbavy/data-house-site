# Data House Website

## Project Overview

Data House is a consulting and implementation business focused on helping companies gain control over data, reporting, processes and decision-making.

The website is not intended to position Data House as a software house, BI agency or development company.

The primary value proposition is solving business problems through data, automation and process understanding.

Technology is a tool, not the product.

---

## Positioning

Data House helps companies:

* gain control over their data
* reduce manual work
* improve reporting
* improve decision-making
* optimize business processes
* connect disconnected systems and information sources

Clients are not buying:

* Power BI
* SQL
* Python
* ETL pipelines
* dashboards

Clients are buying:

* solutions to business problems
* better visibility into their business
* more reliable data
* process improvements
* automation where it makes sense

Always communicate business value before technology.

---

## Target Audience

Primary audience:

* e-commerce companies
* distribution businesses
* small and medium-sized companies

Typical problems:

* manual reporting
* data scattered across multiple systems
* inventory management issues
* lack of visibility into business performance
* repetitive manual work
* uncertainty in decision-making
* inconsistent data quality

Communication should focus on these problems rather than on technical implementation details.

---

## Communication Guidelines

### Preferred Language

Use business-oriented language:

* better decisions
* reliable data
* visibility
* efficiency
* automation
* less manual work
* process improvement
* measurable results
* inventory optimization
* reporting automation

### Avoid Leading With Technology

Do not lead copy with:

* Power BI
* Tableau
* SQL
* Python
* ETL
* Data Pipelines
* APIs
* Data Warehouses

These technologies may be mentioned when relevant, but they should never be the primary message.

The website should communicate outcomes, not tools.

---

## Content Principles

When creating new content:

1. Start with the business problem.
2. Explain the impact.
3. Explain the solution.
4. Mention technology only if necessary.

Preferred structure:

Problem → Solution → Result

Not:

Technology → Features → Technical Details

---

## Case Studies

Case studies should focus on:

* business outcomes
* measurable improvements
* process improvements
* decision-making improvements

Examples:

Good:

* Reduced inventory value by €50,000 while maintaining product availability above 80%
* Reduced manual work through automation
* Improved reporting visibility

Avoid:

* Built a Python application
* Created a Power BI dashboard
* Implemented an ETL pipeline

The result is more important than the technology.

## Translation Rules

Every new content element must have:

* Slovak version
* English version

When adding new content:

1. Update HTML
2. Update translations in assets/js/main.js
3. Verify language switching functionality

Never add Slovak-only content.

---

## Design Rules

Maintain the existing visual style.

Prefer:

* clean layouts
* readability
* clarity
* business-oriented presentation

Avoid:

* flashy animations
* unnecessary complexity
* overly technical visualizations
* technology-first messaging

---

## SEO Guidelines

SEO messaging should align with positioning.

Prefer:

* data consulting
* reporting
* automation
* business processes
* e-commerce operations
* decision-making

Avoid technology-stuffed SEO content.

The website should rank for business value, not only for technology keywords.

---

## Future Development Priorities

Priority order:

1. Real case studies
2. Client results
3. Portfolio examples
4. SEO improvements
5. Additional content pages

Do not add new sections simply to make the website longer.

Every new section must strengthen credibility, positioning or business value.

---

## Decision Rule

When making changes to this website, always ask:

"Does this communicate business value or only technology?"

If the answer is "only technology", rethink the change.
